<task type="auto" tdd="true">
  <name>Task 2: Create model loading, prediction, and feature importance module (ML-03)</name>
  <files>app/ml/predict.py</files>
  <action>Create app/ml/predict.py with:
1. `load_model(model_path='models/quality_model.pkl')` function that uses `joblib.load()` to load the pipeline. Returns the pipeline object. Raises `FileNotFoundError` with clear message if model file missing.
2. `predict_batch(pipeline, input_dict)` function that:
   - Accepts the loaded pipeline and a dict with keys: temperature, moisture, pH, color_score, cooking_time, supplier_origin, dryness_level, visual_inspection
   - Encodes supplier_origin using the same LabelEncoder logic (fit on ['Central Java', 'East Java', 'West Java', 'Yogyakarta', 'Central Kalimantan'] — alphabetical order maps to 0-4)
   - Encodes visual_inspection using: {'poor': 1, 'fair': 2, 'good': 3, 'excellent': 4}
   - Builds feature array in exact order: ['temperature', 'moisture', 'pH', 'color_score', 'cooking_time', 'supplier_origin_encoded', 'dryness_level', 'visual_inspection_encoded']
   - Calls `pipeline.predict_proba()` to get confidence scores
   - Returns dict: `{'grade': predicted_grade, 'confidence': float(max_proba), 'probabilities': {class: prob for each class}}`
   - Grade classes from the model are: ['Grade A', 'Grade B', 'Reject'] (alphabetical from LabelEncoder)
3. `get_feature_importance(pipeline, importance_path='models/feature_importance.json')` function that:
   - Loads feature importance from the JSON file
   - Returns list of dicts: `[{'feature': name, 'importance': value}, ...]` sorted by importance descending
   - Falls back to extracting from pipeline's RandomForest if JSON file missing
4. `generate_recommendation(grade, input_dict)` function that returns rule-based recommendation strings:
   - Grade A: "Excellent quality — suitable for export-grade packaging."
   - Grade B: "Good quality — suitable for domestic market. Consider adjusting moisture and pH for Grade A."
   - Reject: "Batch does not meet quality standards. Review production parameters."
   - Additionally, check specific parameters and append actionable advice:
     - If moisture > 4.0: " Moisture too high — increase drying time."
     - If moisture < 1.5: " Moisture too low — check for over-drying."
     - If pH < 4.5: " pH too acidic — review fermentation process."
     - If pH > 7.0: " pH too alkaline — check raw material quality."
     - If temperature < 105 or temperature > 125: " Temperature outside optimal range (105-125°C)."
</action>
