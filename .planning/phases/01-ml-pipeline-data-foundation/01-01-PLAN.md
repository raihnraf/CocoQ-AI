<task type="auto">
  <name>Task 2: Build synthetic data generator with domain-knowledge rules</name>
  <files>app/ml/generate_data.py</files>
  <action>Create `app/ml/generate_data.py` with a function `generate_batch_data(n_samples=200, output_path="data/coconut_sugar_batches.csv")` that:

1. Generates realistic coconut sugar production data using domain-knowledge rules:
   - **Grade A (~40%)**: temperature 110-120°C, moisture 1.5-3.0%, pH 5.5-6.5, color_score 70-90, cooking_time 80-100min, dryness_level 4-5, visual_inspection in ['excellent', 'good']
   - **Grade B (~35%)**: temperature 105-125°C, moisture 3.0-4.5%, pH 4.5-5.5 or 6.5-7.0, color_score 50-75, cooking_time 70-110min, dryness_level 2-4, visual_inspection in ['good', 'fair']
   - **Reject (~25%)**: temperature <105 or >125°C, moisture >4.5%, pH <4.5 or >7.0, color_score <50, cooking_time <70 or >110min, dryness_level 1-2, visual_inspection in ['fair', 'poor']

2. Uses numpy random with seed=42 for reproducibility

3. supplier_origin: randomly chosen from ['Central Java', 'East Java', 'West Java', 'Yogyakarta', 'Central Kalimantan']

4. Generates a batch_id in format 'B-{YYYY}-{NNN}' (e.g., 'B-2026-001')

5. Creates a production_date column with dates spread over the last 6 months

6. Returns a pandas DataFrame and saves to CSV

7. When run as `__main__`, generates 200 samples and saves to `data/coconut_sugar_batches.csv`

Columns in order: batch_id, production_date, temperature, moisture, pH, color_score, cooking_time, supplier_origin, dryness_level, visual_inspection, grade

Use np.random.normal() with appropriate means/stds within each grade's ranges for realistic distributions. Add small noise to make it feel like real sensor data.</action>
