  <name>Task 1: Add GET /api/stats endpoint for dashboard summary statistics</name>
  <files>app/api/routes.py</files>
  <action>
Add a new `StatsAPI` endpoint to `app/api/routes.py` at route `/api/stats` (GET only).

The endpoint queries the SQLite database and returns:
```json
{
  "total_batches": 42,
  "grade_distribution": {
    "Grade A": {"count": 20, "percentage": 47.6},
    "Grade B": {"count": 15, "percentage": 35.7},
    "Reject": {"count": 7, "percentage": 16.7}
  },
  "reject_rate": 16.7,
  "reject_trend": [
    {"date": "2026-05-15", "reject_rate": 12.5},
    {"date": "2026-05-16", "reject_rate": 18.2},
    ...
  ]
}
```

Implementation details:
1. Create class `StatsAPI(MethodView)` with a `get(self)` method
2. Use `get_db()` to get database connection
3. Query total batches: `SELECT COUNT(*) FROM batches`
4. Query grade distribution: `SELECT predicted_grade, COUNT(*) as cnt FROM predictions GROUP BY predicted_grade`
5. Calculate percentages: `(count / total_batches) * 100`, rounded to 1 decimal
6. Calculate reject rate: `(reject_count / total_batches) * 100`, rounded to 1 decimal
7. Query reject trend (last 7 days): `SELECT DATE(created_at) as date, COUNT(*) as total, SUM(CASE WHEN predicted_grade = 'Reject' THEN 1 ELSE 0 END) as rejects FROM predictions GROUP BY DATE(created_at) ORDER BY date DESC LIMIT 7`
8. For each day in trend, calculate `reject_rate = (rejects / total) * 100` rounded to 1 decimal
9. Handle edge case: if total_batches is 0, return zeros for all stats
10. Register with: `api_bp.add_url_rule('/stats', view_func=StatsAPI.as_view('stats'))`

Add the registration line at the bottom of routes.py alongside existing registrations.
